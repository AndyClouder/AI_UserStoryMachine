# StoryMachine 网页版

StoryMachine的网页版提供了与命令行版本相同的功能，但通过用户友好的Web界面进行操作。

## 功能特点

- 📁 **文件上传**: 支持PRD文档和技术规格文档的上传
- 🤖 **AI驱动**: 使用智谱AI自动生成高质量用户故事
- 📋 **故事展示**: 清晰展示用户故事和验收标准
- 💾 **Markdown下载**: 支持将生成的用户故事下载为Markdown文件
- 🎨 **简洁界面**: Google风格的简洁设计，易于使用

## 快速开始

### 1. 安装依赖

```bash
cd storymachine
uv sync
```

### 2. 启动网页服务器

```bash
uv run python run_web.py
```

### 3. 访问网页界面

在浏览器中打开: http://localhost:8000

## 使用方法

### 步骤1: 上传文档

1. **必需**: 选择PRD文档 (.md 或 .txt 格式)
2. **可选**: 上传技术规格文档
3. **可选**: 填写GitHub仓库地址

### 步骤2: 生成用户故事

点击"开始生成用户故事"按钮，系统将：
- 分析您上传的PRD文档
- 调用智谱AI生成用户故事
- 为每个故事生成详细的验收标准

### 步骤3: 查看结果

系统会显示所有生成的用户故事，包括：
- 故事标题
- 用户故事描述
- 角色
- 目标
- 验收标准

### 步骤4: 下载文件

点击"下载Markdown文件"按钮，将所有用户故事保存为本地Markdown文件。

## 技术架构

### 后端技术栈
- **FastAPI**: 高性能的Python Web框架
- **智谱AI**: AI模型集成，提供智能故事生成
- **Python 3.13+**: 现代Python语言特性

### 前端技术栈
- **HTML5 + CSS3**: 现代Web标准
- **原生JavaScript**: 轻量级客户端交互
- **响应式设计**: 支持各种屏幕尺寸

### 核心功能模块
- `web_simple.py`: 主要Web应用逻辑
- `workflow.py`: 复用命令行版本的故事生成工作流
- `activities.py`: 核心AI处理逻辑

## 与CLI版本的关系

网页版完全复用了命令行版本的核心逻辑：
- ✅ 相同的AI模型调用
- ✅ 相同的故事生成逻辑
- ✅ 相同的验收标准生成
- ✅ 相同的中文本地化支持
- ✅ 相同的智谱AI集成

网页版与CLI版本可以共存使用，互不影响。

## 配置要求

- 智谱AI API密钥 (在环境变量中配置)
- Python 3.13+
- 现代Web浏览器

## 故障排除

### 常见问题

1. **端口占用**: 如果8000端口被占用，请修改 `run_web.py` 中的端口号
2. **依赖缺失**: 确保运行 `uv sync` 安装所有依赖
3. **AI API错误**: 检查智谱AI API密钥是否正确配置

### 日志查看

服务器启动后会在控制台显示详细的运行日志，包括：
- 文件上传状态
- AI处理进度
- 错误信息（如有）

## 开发说明

### 项目结构

```
storymachine/
├── src/storymachine/
│   ├── web_simple.py      # 网页版主应用
│   ├── workflow.py        # 核心工作流逻辑
│   ├── activities.py      # AI处理活动
│   └── ai_zhipuai.py      # 智谱AI集成
├── run_web.py            # 网页版启动脚本
└── WEB_README.md         # 本说明文件
```

### 自定义配置

如需修改服务器配置，请编辑 `run_web.py` 文件中的以下参数：
- `host`: 服务器监听地址
- `port`: 服务器端口号
- `log_level`: 日志级别
